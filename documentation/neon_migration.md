# Neon Postgres Migration (ChromaDB Replacement)

## Date

2026-03-12

## Summary

Replaced ChromaDB (local vector store) with Neon Postgres + pgvector for the interpretive report RAG pipeline. The `mmpi3` schema in Neon already contained 357 embedded chunks from the "Interpreting the MMPI-3" textbook (Ben-Porath & Sellbom), with 384-dim embeddings generated by `sentence-transformers/all-MiniLM-L6-v2`. No data migration was needed.

## Why

- Eliminates the ChromaDB dependency and local worksheet ingestion step
- Reuses the existing Neon infrastructure shared with `researchrag`
- Interpretation context now comes from the authoritative MMPI-3 textbook stored in Neon, not manually created worksheet files

## Database Details

- **Schema:** `mmpi3` (existing, created via `researchrag` topic schema template)
- **Table:** `mmpi3.chunks` â€” 357 rows, each with `content` (text), `chunk_index` (int), `source_type` ("fulltext"), and `embedding` (vector(384))
- **Source:** "Interpreting the MMPI-3" textbook (paper_id=1)
- **pgvector extension:** Installed in `corpus` schema; search path set to `mmpi3, corpus, public`
- **Distance metric:** Cosine distance (`<=>` operator)
